# 🚀 Day 5 – Context Engineering

## Objective

Today I learned how Context Engineering significantly improves AI-generated responses by providing relevant background information instead of using only a generic prompt.

---

# Prompt A (Without Context)

Created a generic 30-day learning roadmap without providing any personal information.

## Output Summary

- Generic roadmap
- Common learning resources
- No personalization
- No career-specific recommendations

---

# Prompt B (With Context)

Provided detailed information including:

- Current Situation
- Skills
- Career Goal
- Experience Level
- Available Time
- Learning Style
- Projects Completed

## Output Summary

- Personalized learning roadmap
- Career-specific recommendations
- Project-based learning
- Practical milestones
- Tailored AI and Software Engineering roadmap

---

# Comparison

## Prompt A

- Generic recommendations
- Assumed beginner level
- No personalized projects
- Broad roadmap

## Prompt B

- Personalized roadmap
- Based on my skills and experience
- Suggested relevant technologies
- Focused on AI Engineering
- Included project-based learning
- More realistic and actionable

---

# Key Learnings

- Better context leads to better AI outputs.
- AI makes fewer assumptions when provided with relevant information.
- Context Engineering is one of the most important Prompt Engineering techniques.
- Personalized prompts generate more useful and practical results.
